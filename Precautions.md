# ⚠️ 프로젝트 개발 주의사항 및 가이드 (Precautions)

이 문서는 프로젝트의 안정성을 유지하고, 팀원 간의 협업 시 발생할 수 있는 충돌을 방지하기 위한 핵심 규칙을 담고 있습니다. 처음 참여하시는 분은 반드시 정독해 주세요.

---

## 1. 아키텍처 규칙: "마음대로 참조하지 마세요"

### 1.1 DI Container (의존성 주입) 활용
- **규칙**: 다른 클래스의 기능을 가져올 때 `FindObjectOfType`이나 `GameObject.Find`를 사용하지 마세요.
- **이유**: 프로젝트가 커지면 찾기 속도가 느려지고 에러를 찾기 힘듭니다.
- **방법**: `DIContainer.Resolve<인터페이스>()`를 사용하세요.
    - 예: 유저 데이터를 쓸 때는 `IDataService data = DIContainer.Resolve<IDataService>();`

### 1.2 싱글톤(Singleton) 금지
- **규칙**: `public static Instance` 형태의 싱글톤을 새로 만들지 마세요.
- **이유**: 멀티플레이 환경에서는 객체가 여러 개 생길 수 있어 충돌의 원인이 됩니다. 모든 전역 서비스는 `DIContainer`에 등록하여 관리합니다.

---

## 2. 데이터 관리: "엑셀이 법입니다"

### 2.1 ScriptableObject(SO) 직접 수정 금지
- **규칙**: `Assets/Resources/Data/Fish` 폴더 안의 물고기 데이터 파일을 유니티 인스펙터에서 직접 수정하지 마세요.
- **이유**: 엑셀(`FishData.xlsx`)에서 데이터를 다시 컨버트하면 직접 수정한 내용이 모두 지워집니다.
- **방법**: 반드시 엑셀 파일을 수정한 뒤, 상단 메뉴의 `Tools > Excel > Convert...` 기능을 실행하세요.

### 2.2 ID값 준수
- **규칙**: 물고기나 아이템의 `ID`는 영문 소문자와 언더바(_)만 사용하세요. (예: `fish_bass`)
- **이유**: 서버 저장 및 이미지 매칭 시 대소문자나 오타로 인해 데이터가 꼬일 수 있습니다.

---

## 3. 멀티플레이 (Mirror): "서버와 클라이언트를 구분하세요"

### 3.1 [Command]와 [TargetRpc]
- **Command**: 클라이언트가 서버에 "나 이거 했어!"라고 요청할 때 사용합니다.
- **TargetRpc**: 서버가 특정 클라이언트에게만 "너 이거 받았어!"라고 결과를 알려줄 때 사용합니다.
- **주의**: 중요 데이터(돈, 경험치, 아이템)는 반드시 **서버(Command)**에서 계산하고 결과만 클라이언트에 전달해야 합니다. 클라이언트에서 돈을 더하는 코드를 짜면 해킹에 취약해집니다.

### 3.2 로컬 플레이어 체크
- **규칙**: 내 캐릭터가 아닌 다른 사람의 캐릭터를 내 키보드로 움직이지 않게 하려면 항상 `if (!isLocalPlayer) return;` 코드를 확인하세요.

---

## 4. 유니티 에디터 설정: "이름과 레이어가 중요합니다"

### 4.1 물(Water) 레이어
- **규칙**: 낚시가 가능한 물 오브젝트는 반드시 `Layer`가 `Water`로 설정되어 있어야 합니다.
- **이유**: 낚시찌가 물에 닿았는지 판단할 때 이 레이어 이름을 기준으로 검사합니다.

### 4.2 자식 오브젝트 이름
- **규칙**: 플레이어 프리팹 내부의 `FishingRope`, `TipPoint`, `HookPoint` 등의 이름을 함부로 바꾸지 마세요.
- **이유**: 코드가 이 이름을 검색해서 낚싯줄을 연결합니다. 이름을 바꿔야 한다면 `FishingPlayer.cs` 코드도 같이 수정해야 합니다.

---

## 5. UI 제작: "TextMeshPro만 사용하세요"

- **규칙**: 유니티 기본 `Legacy Text`는 절대 사용하지 마세요. 모든 텍스트는 `TextMeshPro - Text`를 사용합니다.
- **이유**: 한글 폰트 깨짐 방지 및 고품질 그래픽 유지를 위함입니다.
- **앵커(Anchor) 설정**: 창이 커지거나 작아져도 UI가 찢어지지 않게 앵커 포인트를 구석이나 중앙에 잘 고정하세요.

---

## 6. 자주 발생하는 에러 해결 (FAQ)

| 현상 | 원인 | 해결책 |
| :--- | :--- | :--- |
| **낚시가 안 돼요** | 수면 레이어가 `Water`가 아님 | 수면 오브젝트 레이어를 `Water`로 변경 |
| **물고기 이미지가 안 나와요** | 이미지 이름과 엑셀 ID가 다름 | `Tools > Fish > Match Icons...` 실행 |
| **NullReferenceException** | 오브젝트 연결 누락 | 인스펙터 창에서 빈 칸(None)이 없는지 확인 |
| **서버 접속 후 렉이 걸려요** | 에디터 로그 확인 | 콘솔창에 뜨는 빨간 에러 메시지를 팀장에게 공유 |

---
**기억하세요!** 
모르는 코드를 건드리기 전에 반드시 팀원에게 물어보고, 수정 후에는 `Console` 창에 빨간 에러가 뜨지 않는지 확인하는 습관이 제일 중요합니다.
